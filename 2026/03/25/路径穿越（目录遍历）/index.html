路径穿越（目录遍历）

简单来说我们一般在通过文件包含漏洞打入服务器时，一般都会遇到目录过滤的问题，他会过滤掉我们对于某一个目录的直接访问，因此我们需要通过路径穿越的过滤漏洞来达成我们访问目标的目的。

一般CTF应用：

原理：

路径穿越 = 用 ../ 跳出当前目录，访问服务器上的其他文件

  • ../ = 返回上一级目录
  • 拼很多 ../ 就能一路跳到根目录
  • 然后直接访问 flag.php / f1a9 / flag 这类文件

f1a9的作用

1、规避关键词过滤，CTF中经常出现WAF过滤的问题，有可能直接过滤flag或flag.php。

用 f1a9 这种字母 + 数字的混淆写法，可以绕过这类简单的关键词检测，让你在构造 payload 时不会被 WAF 或后端过滤规则拦下。

2、真实文件名被隐藏了

命题人不想让你一眼就猜到目标文件名：

  • 如果直接叫 flag.php，新手也能立刻猜到去读这个文件
  • 改成 f1a9（看起来像随机字符串），会增加你寻找目标文件的难度
  • 你需要通过目录扫描、报错信息或其他线索才能发现这个文件
